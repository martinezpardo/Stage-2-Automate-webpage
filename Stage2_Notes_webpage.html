1. The Basics of the Web and HTML:

The Basics of the Web

The web (www): collection of html documents (HyperText Markup Language). HTML is what glues the Internet together. Around 30 billion pages

Main pieces: you, your computer + browser (a program to display files found on the web), the internet (the world largest computer network), http (the main protocol of the web), servers (computer in a closer that host the files)

HTML (Hyper Text markup language): is made up of text content (what you see), markup (what it looks like), references to other documents (e.g.: images + videos), links to other pages

Computers are Stupid: Programmers need to write exactly the way a computer understands (also known as writing with correct "syntax").

Abstract thinking to avoid repetition: Abstract thinking means finding similarity, or as programmers would say, generality amongst seemingly different things. It would be impractical for a programmer to specifically program each of these pages individually. Abstraction allows to avoid unnecessary repetition of work.

Programmers Can't Remember Everything: There are too many details to keep everything in your head.

Basics of HTML

Basic HTML Vocabulary:

  • Tag: An HTML tag is always contained within angled brackets. Most tags have an opening tag (<p> for example) and a closing tag, (</p>). Some tags (called "void" tags) do not require a closing tag (like the <br> tag).
  • Element: An HTML element refers to everything within a set of opening and closing tags.
  • Attribute: This is a property of an HTML element. For example, to set thehref attribute of an anchor tag to the Udacity URL, you would write<a href="www.udacity.com">

HTML Structure: there are many tags. An opening for the type of document !DOCTYPE HTML>. Then HTML opening html>. Then head (metadata, title, etc.). then body.
Whitespace: all blank in html translates to a space. For a new line: br break (void). It's an inline element. Br ends a line
Paragraph tag: It's a block element. It's rendering an invisible box around the text. There are also container elements
Span (inline) / Div (block): they just contain the content, and you can add styles to them

2. Creating a structured document with HTML:

Intro to HTML and CSS

How do you translate a design to a website?
Think of a website like a house:

  • HTML: structure
  • CSS: style (decoration, colors,...)
  • Javascript: interactive components (remote control, garage door open, etc...)
  • Tools to build it

HTML: language of the web, specific syntax. Basic word is a tag. The browser turns the tags into elements that form a tree (HTML Tree-like structure or branching structure)
All visible elements are rectangular on a webpage, all visible text is on the code. HTML elements are either inline or block.
It does this thanks to Document Object Model (standard convention). Each HTML creates an element in the DOM that the browser uses to display the page. It can be empty, content or another element. Each tag has a name or type. Each tag can also have attributes with values.

CSS: another language. Allows to use specific syntax and rules to change how elements look on the page: font, size, background, position, color, etc.

To tie HTML and CSS: the HTML document can provide the browser with information on what style to use and where to get it.

Everything is a box in a webpage: Boxify a Design. Breakdown any design into boxes. Move from big to the small. <div> don't only contain the content, it creates a box around the content, to separate boxes. Add class names to identify them.

3. Adding CSS Style to HTML Structure:

Basics of CSS

CSS stands for "Cascading Style Sheets"
You can have several CSS files referenced in an HTML file. Most specific rule applies (Cascading)
Style can be added on the same HTML file in between style tags and even inline (not a good idea)

Inheritance: is a key feature in CSS, it relies on the ancestor-descendant relationship to operate. Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants. It relies on the document tree. In general, descendant elements inherit text-related properties, but box-related properties are not inherited.
In CSS you first write a selector, to identify what the style will be applied to. Another way is to select elements by class (attribute, column, value).
CSS properties: https://developer.mozilla.org/en-US/docs/Web/CSS/Reference
Changing the style properties of an element only changes how the element is displayed, not the element itself

Box Sizing:

  1. HTML elements are boxes and each box has 4 components
  2. Because there are so many components to each box, it can often be hard to get the size of a box just right
  3. There are two techniques you can use to help deal with sizing issues:
    • Set sizes in terms of percents rather than pixels
    • Set the box-sizing attribute to border-box for every element
  5. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently

Box Positioning:
- Divs are block elements (as opposed to inline), so by default they take up the entire width of a page
- Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other

Other takeaways

Code - test - refine:

  1. Look for natural boxes
  2. Look for repeated styles and semantic elements (relate to HTML)
  3. Write HTML
  4. Apply styles: from biggest to smallest
  5. Fix things (resize windows, other browsers, etc.)

In HTML, code comments begin with greater!-- and end with -->. Everything in between is ignored. In CSS, code comments begin with /* and end with */.

3. Input -> Function -> Output:

What is a function (procedure)?

Functions take input, do something with it, and then produce output.

Phyton syntax to create functions

Procedure (input,input,...) Inputs are also called operands, arguments. Inputs must match the ones used in the procedure. Return is the output. If you want a function to produce output, it must end with a return statement.
First line of a function: must begin the line with def (lowercase). After def there must be a function name. Next, it must have a set of parentheses with the required parameters inside. The line must end with : colon.
In the body of the function: every line in the function must be indented.

def procedure(x):
answer = x * x
return answer
Using functions

Once created, you can use the function by writings its name followed by () with the required inputs. This helps avoid repetition, as instead of writing the definition everytime we need it we just define it once and then use it by simply writing its name and inputs.
